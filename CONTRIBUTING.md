# Contributing to AgentWatchdogPro

Thanks for helping improve AI agent safety.

## How to contribute

1. Fork the repo
2. Create a branch: `feature/<name>` or `fix/<name>`
3. Add tests for any security/risk rule changes
4. Open a PR with:
   - what changed
   - why it matters
   - before/after behavior

## Good first contributions

- New detection rules (with test cases)
- False-positive reduction for existing rules
- SDK examples (Python/Node)
- Dashboard UX improvements for analyst workflows
- Documentation and attack scenario playbooks

## PR checklist

- [ ] Code compiles
- [ ] Tests pass
- [ ] Added/updated test for rule changes
- [ ] No secrets or credentials committed
- [ ] README/docs updated if behavior changed

## Security notes

- Please do not publish zero-day exploit details in public issues.
- For sensitive disclosures, open a private security report first.
